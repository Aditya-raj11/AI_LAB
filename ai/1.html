Section 01
Bayes' Theorem
Definition
Bayes' Theorem describes the probability of an event based on prior knowledge of conditions related to it. It is the foundation of Bayesian inference — updating beliefs when new evidence is observed.
P(A | B) = [ P(B | A) × P(A) ] / P(B)
P(A|B) = Posterior  ·  P(B|A) = Likelihood  ·  P(A) = Prior  ·  P(B) = Evidence (Marginal Probability)
Prior — P(A)
Our belief about A before observing B. Represents domain knowledge.
Likelihood — P(B|A)
How probable is evidence B if A is true? Comes from data or domain knowledge.
Posterior — P(A|B)
Updated belief about A after observing B. This is the final answer.
P(B) = P(B|A)·P(A) + P(B|¬A)·P(¬A)    [Total Probability]
Denominator ensures the posterior is a proper probability (sums to 1)

Medical Disease Detection

A rare disease affects 1% of the population. A diagnostic test has sensitivity 99% and specificity 95%.

Step 1 — Compute P(T⁺) using Total Probability

P(T⁺) = P(T⁺|D)·P(D) + P(T⁺|¬D)·P(¬D) = (0.99)(0.01) + (0.05)(0.99) = 0.0099 + 0.0495 = 0.0594

Step 2 — Apply Bayes' Theorem

P(D|T⁺) = P(T⁺|D)·P(D) / P(T⁺) = (0.99 × 0.01) / 0.0594 ≈ 0.1667 ≈ 16.67%

Section 02
Bayesian / Belief Networks (BBN)
Definition
A Bayesian Belief Network (BBN) is a Directed Acyclic Graph (DAG) where each node is a random variable with a Conditional Probability Table (CPT). Edges encode direct probabilistic dependencies.
P(X₁, X₂, ..., Xₙ) = ∏ᵢ P(Xᵢ | Parents(Xᵢ))
Joint Distribution = Product of local conditional probabilities — exponentially more compact than full joint table
📌 CPT Size: A node with k binary parents needs 2ᵏ rows. A full joint table for n binary variables needs 2ⁿ rows. BBN saves memory through conditional independence.

Section 03
Markov Models
The Markov Property
A stochastic process has the Markov Property if the next state depends only on the current state, not on the history.
P(Xₜ₊₁ | X₁,...,Xₜ) = P(Xₜ₊₁ | Xₜ)
"The future is independent of the past, given the present."

Key Components of a Markov Chain

Formal Definition

A Markov Chain is defined by: λ = (S, A, π)

  • S = Finite set of states
  • A = Transition probability matrix
  • π = Initial state distribution

Constraints

A[i][j] ≥ 0  &&  Σⱼ A[i][j] = 1
(Each row of A sums to 1)

State Evolution

πₜ = π₀ · Aᵗ
State distribution at time t = initial × matrix power

Stationary Distribution

π* satisfies: π* = π* · A
Long-run fraction of time spent in each state.

Section 04
Hidden Markov Models (HMM)
Definition
A Hidden Markov Model (HMM) is a doubly stochastic process: an underlying hidden Markov chain drives a sequence of observable emissions. We cannot see the states — only the observations.

Formal Parameters: λ = (S, V, A, B, π)

HMM Components Summary
SymbolNameDescription
SState SpaceHidden states {s₁, s₂, ..., sN}
VObservation AlphabetObservable symbols {v₁,...,vM}
ATransition MatrixA[i][j] = P(Xₜ₊₁=sⱼ|Xₜ=sᵢ)
BEmission MatrixB[i][k] = P(Oₜ=vₖ|Xₜ=sᵢ)
πInitial Distributionπ[i] = P(X₁=sᵢ)
Solid = transition (A), Dashed = emission (B)

Three Fundamental Problems of HMM

Problem 1 — Evaluation
Given: λ, O
Find: P(O|λ)
Algorithm: Forward (α-pass)
Time: O(N²T)
Problem 2 — Decoding
Given: λ, O
Find: Best state seq Q*
Algorithm: Viterbi (DP)
Time: O(N²T)
Problem 3 — Learning
Given: O
Find: Best λ* = (A*,B*,π*)
Algorithm: Baum-Welch (EM)
Time: Iterative

The Forward Algorithm — Evaluation Problem

Purpose
Computes P(O | λ) — the probability of observing a sequence O given the model. Used for model comparison and likelihood scoring.
αₜ(i) = P(O₁,...,Oₜ, Xₜ=sᵢ | λ)
α_t(i) = joint probability: observed O₁...Oₜ AND are in state i at time t
Forward Algorithm Steps
StepFormulaDescription
Initialisationα₁(i) = π[i] · B[i][O₁]t=1: prior × emission
Recursion (t=2..T)αₜ(j) = [Σᵢ αₜ₋₁(i)·A[i][j]] · B[j][Oₜ]Sum over all previous states
TerminationP(O|λ) = Σᵢ αT(i)Sum over all final states
